搭建网站遇到的问题

作者：ZY 围观群众：0 更新于

1 12月 2023

错误：npm: command not found：

在我搭建博客的时候。我已经安装了node.js和git bash，准备使用npm来安装Hexo。然而，当我在bash中输入npm install -g hexo-cli时，我却得到了一个错误提示：npm: command not found。

我感到非常困惑，因为我明明已经安装了node.js，npm应该是自带的。我开始在网上搜索这个问题，发现可能是因为npm没有被添加到环境变量中，导致bash找不到npm命令。

于是，我打开了环境变量设置，检查了一下PATH变量。果然，我发现npm的路径并没有在PATH变量中。我立即将npm的路径添加到PATH变量中，然后重新打开了bash，再次输入了npm install -g hexo-cli。这次，命令成功执行了，Hexo也被成功安装。


错误：Hexo的网页本地正常，但是部署到网页上不正常

在本地预览时，一切都看起来很完美，主题、图片和样式都显示得很好。然而，当我将我的博客部署到GitHub Pages上时，问题出现了。

我打开了我的GitHub Pages网站，发现页面上的主题、图片和样式都无法正常显示。我感到非常困惑，因为在本地预览时，一切都是正常的。

我开始在网上搜索这个问题，发现可能是因为我在_config.yml文件中设置的url地址不正确。我检查了一下我的_config.yml文件，发现我设置的url地址确实是错误的。

于是，我将url地址改为了正确的GitHub Pages的地址，然后重新生成和部署了我的博客。这次，当我打开我的GitHub Pages网站时，一切都显示得很好，就像在本地预览时一样。

通过这次经历，我明白了在使用Hexo和GitHub Pages搭建博客时，需要确保_config.yml文件中的设置是正确的。


错误：端口被占用

我已经完成了所有的设置，准备在本地预览我的博客。然而，当我在终端输入hexo server命令时，我得到了一个错误提示：Error: listen EADDRINUSE :::4000。这意味着端口4000已经被其他程序占用。

我感到非常困惑，因为我并没有运行其他的程序。我开始在网上搜索这个问题，发现可能是因为我之前运行的某个程序没有正确关闭，导致端口4000仍然被占用。

于是，我打开了任务管理器，查看了一下正在运行的程序。果然，我发现有一个我之前测试的程序还在运行，而且它正好占用了端口4000。我立即结束了这个程序，释放了端口4000。

然后，我再次在终端输入了hexo server命令。这次，命令成功执行了，我可以通过localhost:4000来预览我的博客了。


错误：npm 命令无法找到

有一天，我正在使用Hexo和GitHub Pages搭建我的个人博客。我已经安装了node.js，准备使用npm来安装Hexo。然而，当我在终端输入npm install -g hexo-cli时，我却得到了一个错误提示：npm: command not found。

我感到非常困惑，因为我明明已经安装了node.js，npm应该是自带的。我开始在网上搜索这个问题，发现可能是因为npm没有被添加到环境变量中，导致终端找不到npm命令。

于是，我打开了环境变量设置，检查了一下PATH变量。果然，我发现npm的路径并没有在PATH变量中。我立即将npm的路径添加到PATH变量中，然后重新打开了终端，再次输入了npm install -g hexo-cli。这次，命令成功执行了，Hexo也被成功安装。
